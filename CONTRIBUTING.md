## Contributing

Thank you for your interest in contributing to serctl! All contributions are welcome, from bug fixes to new features.

### How to contribute

1. Fork the repository
2. Create a new branch for your changes:
```bash
git checkout -b feature/your-feature-name
```
3. Make your changes
4. Commit your changes:
```bash
git commit -m "Add: brief description of your changes"
```
5. Push to your fork:
```bash
git push origin feature/your-feature-name
```
6. Open a Pull Request to the `main` branch

All Pull Requests will be reviewed before being merged.

### Issues

Feel free to open an issue if you find a bug or have a feature request. You can also pick up any open issue and submit a Pull Request with a fix.

### Guidelines

- Keep your changes focused and minimal
- If you are adding a new feature, consider opening an issue first to discuss it
- Follow the existing code style of the project
- Test your changes before submitting
